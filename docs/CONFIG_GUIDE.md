# Project Configuration Guide

## Root-Level Configuration Files

This document explains all configuration files at the project root.

### Essential Files (Keep)

| File | Purpose |
|------|---------|
| `package.json` | Node.js dependencies and scripts |
| `composer.json` | PHP dependencies and Laravel scripts |
| `tsconfig.json` | TypeScript configuration |
| `tailwind.config.js` | Tailwind CSS settings |
| `vite.config.js` / `vite.config.ts` | Vite build configuration |
| `eslint.config.js` | JavaScript/TypeScript linting rules |
| `.prettierrc` | Code formatting rules |
| `phpunit.xml` | PHPUnit/Pest testing configuration |
| `pint.json` | Laravel Pint code style configuration |
| `.env.example` | Environment variables template |

### Development Tools (Keep)

| File | Purpose |
|------|---------|
| `opencode.json` | OpenCode AI integration config |
| `boost.json` | Laravel Boost configuration (skills, guidelines) |
| `AGENTS.md` | AI Agent documentation and prompts |
| `GEMINI.md` | Gemini AI integration guide |

### Design Assets (Keep)

| File | Purpose |
|------|---------|
| `colors.json` | Application color palette (Rose Dusk theme) |
| `modifiction.png` | Visual mockup/screenshot |

### Documentation (Keep)

| File | Purpose |
|------|---------|
| `README.md` | **Main project documentation** |
| `CONTRIBUTING.md` | **Contribution guidelines** |
| `modification.md` | Feature/improvement backlog |
| `analys.md` | Project analysis notes |
